# 等保三级主机加固实施方案

## 1. 前言

### 项目背景

依据《中华人民共和国网络安全法》及国家网络安全等级保护制度要求，为落实 **信息安全等级保护三级（等保三级）** 合规要求，特制定本主机加固实施方案。

等保三级安全计算环境是等保三级系统的核心安全域，覆盖**身份鉴别、访问控制、安全审计、入侵防范、恶意代码防范、资源控制、数据完整性、数据保密性、数据备份恢复、剩余信息保护、个人信息保护、软件容错、集中管控**共计 13 个控制点。本方案旨在通过对操作系统、数据库及核心应用系统实施标准化、可验证的安全加固，使其满足 GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》三级安全计算环境的各项技术指标。

本方案同时参考国际主流的安全基线标准（CIS Benchmarks）及主流厂商的官方安全加固指南，形成"国家标准 + 国际基准 + 厂商规范"三重引用的结构化加固体系。

## 2. 三重引用框架说明

本方案每条加固项均标注以下三重引用信息：

| 引用层次 | 标准/规范 | 标注格式 | 说明 |
|---------|----------|---------|------|
| **第一重** | GB/T 22239-2019 三级 | `[等保三级] 控制点号.控制项名` | 国家等级保护三级要求的对应控制点 |
| **第二重** | CIS Benchmark | `[CIS] 章节号.章节名` | CIS 安全基准的对应章节编号及建议 |
| **第三重** | 操作命令/路径 | `` `具体命令或配置路径` `` | 可执行的加固操作或配置文件路径 |

### 示例

```markdown
### 2.1 口令复杂度策略

**[等保三级]** 身份鉴别(S) — 应对登录用户进行身份标识和鉴别  
**[CIS]** 1.1.1 Ensure password expiration is 365 days or less  
**操作**: `chage -M 90 <user>` / 配置 `/etc/login.defs` 中 `PASS_MAX_DAYS=90`
```

### 控制点代码说明

| 代码 | 含义 | 所属类别 |
|------|------|---------|
| S | 安全功能要求（Security） | 技术要求 |
| G | 通用安全要求（General） | 通用要求 |
| A | 增强要求（Augmented） | 扩展要求 |

## 3. 标准引用说明

### 3.1 GB/T 22239.1-2019

**《信息安全技术 网络安全等级保护基本要求》第1部分：安全通用要求**

- 发布机构：国家市场监督管理总局、中国国家标准化管理委员会
- 实施日期：2019-12-01
- 三级系统安全计算环境覆盖 13 个控制点：
  - **身份鉴别**（S）— 7 个要求项
  - **访问控制**（S）— 9 个要求项
  - **安全审计**（G）— 5 个要求项
  - **入侵防范**（G）— 5 个要求项
  - **恶意代码防范**（G）— 2 个要求项
  - **资源控制**（A）— 4 个要求项
  - **数据完整性**（S）— 3 个要求项
  - **数据保密性**（S）— 3 个要求项
  - **数据备份恢复**（A）— 3 个要求项
  - **剩余信息保护**（S）— 2 个要求项
  - **个人信息保护**（S）— 2 个要求项
  - **软件容错**（A）— 2 个要求项
  - **集中管控**（G）— 6 个要求项

### 3.2 GB/T 28448-2019

**《信息安全技术 网络安全等级保护测评要求》**

- 发布机构：国家市场监督管理总局、中国国家标准化管理委员会
- 实施日期：2019-12-01
- 针对三级系统的**测评要求**，规定了等保三级系统的安全测评方法、测评内容和测评判定准则
- 本方案中的每项加固措施均对应 GB/T 28448-2019 中的测评单元，确保**可测、可验证**

### 3.3 CIS Benchmarks

**Center for Internet Security Benchmarks**

CIS（互联网安全中心）发布的操作系统和应用安全配置基准，是全球最广泛采用的**安全基线标准**之一。本方案参考的 CIS Benchmark 包括：

| 基准名称 | 版本 |
|---------|------|
| **CIS Red Hat Enterprise Linux 9 Benchmark** | v3.0.0 |
| **CIS Windows Server 2022 Benchmark** | v2.0.0 |
| **CIS Oracle Database 19c Benchmark** | v2.0.0 |
| **CIS Ubuntu Linux 22.04 LTS Benchmark** | v2.0.0 |
| **CIS Docker Benchmark** | v1.6.0 |
| **CIS Kubernetes Benchmark** | v1.9.0 |

CIS 评分等级：
- **Level 1**（L1）：基本安全配置，对业务影响极小
- **Level 2**（L2）：增强安全配置，适用于高安全环境

### 3.4 厂商基线

各软硬件厂商发布的官方安全加固指南：

| 厂商 | 基线名称 | 适用范围 |

| Red Hat | Red Hat Security Guide / SCAP Security Guide | RHEL / CentOS |
| Oracle | Oracle Database Security Guide / Oracle CIS 加固 | Oracle 数据库 |
| SUSE | SUSE Linux Enterprise Security Guide | SLES |

## 5. 使用说明

### 5.1 适用对象

- 等保三级系统的**系统管理员**（操作系统加固）
- **数据库管理员**（数据库加固）
- **应用运维人员**（应用系统加固）
- **安全管理员**（加固验证与审计）
- **测评机构人员**（合规检查对照）

### 5.2 前置条件

在执行加固操作前，请确认以下条件：

1. **备份确认**
   - ✅ 系统全量备份已完成
   - ✅ 数据库数据文件备份已完成
   - ✅ 应用程序配置文件备份已完成
   - ✅ 系统快照（虚拟机环境）已创建

2. **测试环境**
   - ✅ 加固方案已在测试环境验证
   - ✅ 业务功能回归测试通过
   - ✅ 回退方案已就绪

3. **变更管理**
   - ✅ 变更申请已审批
   - ✅ 变更窗口已确定
   - ✅ 回退预案已就绪

### 5.3 执行流程

```
┌─────────────────────────────────────┐
│        1. 安全评估与差距分析            │
│     (对照三重标准识别未达标项)          │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│        2. 测试环境验证                │
│     (在测试环境执行加固+业务测试)       │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│        3. 生产环境加固                │
│     (按操作系统→数据库→应用顺序执行)    │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│        4. 加固验证                    │
│     (运行检测脚本+人工抽查)            │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│        5. 合规自评                    │
│     (对照自检清单逐项确认)             │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│        6. 持续监控                    │
│     (定期审计+安全基线巡检)            │
└─────────────────────────────────────┘
```

### 5.4 执行优先级建议

| 优先级 | 加固项类别 | 说明 |
|--------|-----------|------|
| **P0** | 身份鉴别、访问控制 | 基础安全要求，必须先完成 |
| **P1** | 安全审计、入侵防范 | 检测类防御，需在基础加固后配置 |
| **P2** | 数据加密、备份恢复 | 数据保护，需要前期配置完成 |
| **P3** | 资源控制、集中管控 | 增强型加固，可在后续迭代完成 |

### 5.5 自动化工具使用

**Linux 加固检测脚本**：

```bash
# 运行 Linux 安全检测
chmod +x scripts/audit_scan.py
python3 scripts/audit_scan.py --os linux --output report.html

# 一键部署 Linux 加固
bash scripts/deploy.sh --os linux --apply

# 仅生成加固报告（不实际修改系统）
bash scripts/deploy.sh --os linux --check-only
```

**Windows 加固检测脚本**（需要管理员权限）：

```powershell
# 运行 Windows 安全检测
.\scripts\windows_check.ps1 -OutputPath .\report.html
```

**Oracle 加固检测脚本**：

```sql
-- 连接数据库后运行
@scripts/oracle_check.sql
```

### 5.6 版本管理

- 本方案遵循语义化版本（SemVer）：`主版本.次版本.修订号`
- 主版本变更：标准版本升级或架构调整
- 次版本变更：新增加固项或控制点
- 修订号变更：修正错误或优化描述
