# 交付物三：评分标准

> **结论先行**：三级等保评分采用「权重赋值×符合度系数」模型，满分 100 分。单个控制点不合格即判定该层面不合格（一票否决），但整改优先级按「权重×差距度」矩阵排序。本标准源自 GB/T 22239-2019 + 权重赋值表 + GB/T 28448-2019 测评判定准则。

---

## 1. 核心评分模型

### 1.1 评分公式

```
控制点评分 = Σ(要求项得分) / 要求项总数 × 控制点权重系数
安全层面评分 = Σ(控制点评分) / Σ(控制点权重系数) × 100
整体系统评分 = Σ(所有层面评分 × 层面权重调整) / 层面数
```

### 1.2 符合度系数（三档评分）

| 评分档 | 符号 | 分值系数 | 判定标准（GB/T 28448-2019） |
|:---|:---:|:---:|:---|
| 符合 | ✅ | 1.0 | 所有子项均为肯定，无偏离 |
| 部分符合 | 🔶 | 0.5 | 主要项符合，但存在 1-2 项次要偏离 |
| 不符合 | ❌ | 0.0 | 存在关键项不符合，或偏离项 ≥3 |

> 注意：三级系统中，身份鉴别(S3)、访问控制(S3) 的某些关键要求项（如强制访问控制SELinux、双因素鉴别）是**一票否决型**——该项不符合即使总分达标，该控制点也判定为不符合。

### 1.3 权重赋值表（三级S3A3G3）

权重分为三档：

| 权重 | 含义 | 示例控制点要求项 |
|:---:|:---|:---|
| **1.0** | 关键项 | 电子门禁、冗余供电、备用供电系统、电磁屏蔽、路由控制、边界隔离、审计覆盖、入侵检测、身份鉴别双因素等 |
| **0.5** | 重要项 | 物理访问控制、防雷击、温湿度控制、网络审计、恶意代码防范、网络设备防护、管理员权限分离、口令复杂度等 |
| **0.2** | 一般项 | 防震选址、通信线缆铺设、介质管理、防水防潮、电力电缆铺设、电磁接地等 |

## 2. 符合率计算示例

### 2.1 示例场景

某 Linux 服务器（G1）身份鉴别控制点（7 项），检查结果：

| 要求项 | 权重 | 检查结果 | 得分 |
|:---:|:---:|:---:|:---:|
| a.用户标识唯一性 | 1.0 | ✅ 符合 | 1.0 |
| b.登录身份标识鉴别 | 1.0 | ✅ 符合 | 1.0 |
| c.双因素鉴别 | 1.0 | ❌ 不符合 | 0.0 |
| d.口令复杂度 | 0.5 | 🔶 部分符合(password minlen=8 < 14) | 0.25 |
| e.登录失败处理 | 1.0 | ✅ 符合 | 1.0 |
| f.超时自动退出 | 0.5 | ✅ 符合 | 0.5 |
| g.鉴别警示 | 0.5 | ❌ 不符合 | 0.0 |

**身份鉴别控制点评分计算：**

```
实际加权得分 = 1.0+1.0+0.0+0.25+1.0+0.5+0.0 = 3.75
最高加权得分 = 1.0+1.0+1.0+0.5+1.0+0.5+0.5 = 5.50
符合率 = 3.75 / 5.50 = 68.18%
```

### 2.2 整体安全计算环境评分

假设 Linux 服务器 10 个控制点的检查结果：

| 控制点 | 加权最高 | 加权实际 | 符合率 |
|:---|:---:|:---:|:---:|
| 身份鉴别 | 5.50 | 3.75 | 68.2% |
| 访问控制 | 4.00 | 3.50 | 87.5% |
| 安全审计 | 3.50 | 3.00 | 85.7% |
| 入侵防范 | 3.00 | 2.50 | 83.3% |
| 恶意代码防范 | 1.50 | 1.50 | 100% |
| 资源控制 | 2.00 | 1.50 | 75.0% |
| 数据完整性 | 2.00 | 1.00 | 50.0% |
| 数据保密性 | 2.00 | 2.00 | 100% |
| 剩余信息保护 | 1.00 | 0.50 | 50.0% |
| 集中管控 | 4.00 | 2.00 | 50.0% |
| **合计** | **28.50** | **21.25** | **74.6%** |

**结论**：该服务器安全计算环境整体符合率约 **74.6%**，仍需整改的主要薄弱点为：
1. 身份鉴别（68.2%）— 缺少双因素鉴别
2. 集中管控（50.0%）— 未接入堡垒机和日志中心
3. 数据完整性（50.0%）— 未部署文件完整性检测
4. 资源控制（75.0%）— 缺乏磁盘/CPU阈值告警

## 3. 整改优先级矩阵

### 3.1 优先级 = 权重 × 差距度

```
差距度 = 1 - 当前符合率
优先级分数 = 权重档位 × 差距度
```

| 优先级等级 | 判定条件 | 行动要求 | 完成时限 |
|:---|:---|:---|:---|
| **P0 — 紧急** | 权重=1.0 且 符合率<50% | 立即整改，影响合规判定 | 1周内 |
| **P1 — 高** | 权重=1.0 且 符合率50-80% 或 权重=0.5 且 符合率<50% | 限期整改，纳入月计划 | 2周内 |
| **P2 — 中** | 权重=0.5 且 符合率50-80% 或 权重=0.2 且 符合率<50% | 纳入季度整改计划 | 1月内 |
| **P3 — 低** | 权重=0.2 且 符合率≥50% 或 权重=0.5且符合率>80% | 持续改进 | 下一周期 |

### 3.2 一票否决项（必须符合）

以下项如果不符合，即使其他项满分，该控制点也判定为不符合：

| # | 控制点 | 一票否决项 | 依据 |
|:---:|:---|:---|:---|
| 1 | 身份鉴别 | c)双因素鉴别（三增强级） | GB/T 28448-2019 三级判定规则 |
| 2 | 身份鉴别 | d)口令复杂度（符合或显著偏离判定） | 综合判定关键指标 |
| 3 | 访问控制 | b)强制访问控制（SELinux/AppArmor） | 三级安全计算环境核心要求 |
| 4 | 安全审计 | a)审计系统是否安装运行 | 基本合规门槛 |
| 5 | 入侵防范 | a)最小安装原则（关闭不必要端口/服务） | 基础安全底线 |
| 6 | 恶意代码防范 | a)防病毒软件安装运行 | 基本合规门槛 |
| 7 | 数据备份恢复 | a)关键数据有备份机制 | 等级保护核心要求 |

## 4. 综合判定等级

根据 GB/T 28448-2019，系统整体安全等级判定：

| 安全计算环境符合率 | 整体判定 | 含义 |
|:---:|:---|:---|
| ≥ 85% | **基本符合** | 核心控制在符合范围内，部分一般项需改进 |
| 70%-84% | **存在风险** | 关键控制点存在偏离，需限期整改 |
| < 70% | **不符合** | 不满足三级等保要求，必须全面整改 |

> **重要**：即使总分 ≥ 85%，若存在一票否决项不符合，整体判定仍为**不符合**。

---

## 5. ISEAA 正式报告评分对照

ISEAA 官方测评报告不使用百分比评分，而是采用**符合 / 部分符合 / 不符合**的三档判定。本项目评分标准作为**整改实施过程评估工具**，最终以测评机构出具的正式报告为准。两者对应关系：

| 本项目评分 | ISEAA 判定 | 说明 |
|:---:|:---|:---|
| 控制点符合率 ≥ 85% | 符合 | 所有子项均为肯定 |
| 控制点符合率 50%-84% | 部分符合 | 存在次要偏离但不影响整体 |
| 控制点符合率 < 50% | 不符合 | 关键项偏离或多数项不符合 |
