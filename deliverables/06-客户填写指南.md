# 交付物六：客户填写指南

> **结论先行**：资产采集是等保整改的起点，也是测评失败的第一大原因。本指南将「给客户填的表」从专业术语翻译成白话，附填写示范和易错点标注，确保建设单位能一次性提交有效数据。

## 2. 逐字段填写说明

### 2.1 系统名称

```
┌──────────────────────────────────────────────────────────────┐
│ 📝 填写说明                                                   │
│                                                              │
│ 系统名称 = 该服务器承载的业务系统的名称                        │
│                                                              │
│ ✅ 正确示例：                                                  │
│   • 医院HIS信息系统                                           │
│   • 综合办公OA系统                                            │
│   • 电子病历系统（EMR）                                        │
│                                                              │
│ ❌ 常见错误：                                                  │
│   • "192.168.1.10" ← 这是IP，不是系统名称                      │
│   • "Web服务器" ← 太笼统，请注明是哪个系统的Web                  │
│   • "测试系统1" ← 请写全称                                    │
│   • 留空 ← 必须填写                                           │
│                                                              │
│ 💡 提示：如一台服务器运行多个系统，请拆分为多行填写               │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 系统IP地址

```
┌──────────────────────────────────────────────────────────────┐
│ 📝 填写说明                                                   │
│                                                              │
│ 填写服务器的管理IP地址或业务IP（主IP）                          │
│                                                              │
│ ✅ 正确示例：                                                  │
│   • 192.168.1.10                                              │
│   • 10.0.0.5                                                  │
│                                                              │
│ ❌ 常见错误：                                                  │
│   • 留空 → 案例报告就是被这个坑了，IP为空等于资产不存在           │
│   • "192.168.1" ← 不完整，缺少最后一段                         │
│   • "127.0.0.1" ← 这是本地回环，没有任何意义                    │
│   • 写多个IP用分号→只写主IP，多个IP备注里写                     │
│                                                              │
│ 💡 提示：如果不知道IP，请用以下命令查看                          │
│   Windows: ipconfig → IPv4 地址                               │
│   Linux: ip addr → inet 那行                                   │
└──────────────────────────────────────────────────────────────┘
```

### 2.3 操作系统版本

```
┌──────────────────────────────────────────────────────────────┐
│ 📝 填写说明                                                   │
│                                                              │
│ 填写完整的操作系统名称和版本号                                  │
│                                                              │
│ ✅ 正确示例：                                                  │
│   • CentOS Linux release 7.9.2009 (Core)                     │
│   • Windows Server 2019 Standard                             │
│   • Ubuntu 20.04.6 LTS                                       │
│   • Red Hat Enterprise Linux 8.6 (Ootpa)                     │
│                                                              │
│ ❌ 常见错误：                                                  │
│   • "Linux" ← 太笼统，不知道什么发行版                          │
│   • "Windows" ← 不知道是Win7/Win10/Server哪个                  │
│   • "CentOS" ← 缺少小版本号，无法对应加固方案                    │
│                                                              │
│ 💡 查看命令：                                                  │
│   Linux: cat /etc/os-release 或 hostnamectl                   │
│   Windows: winver 或 msinfo32                                 │
└──────────────────────────────────────────────────────────────┘
```

### 2.4 主机名

```
┌──────────────────────────────────────────────────────────────┐
│ 📝 填写说明                                                   │
│                                                              │
│ 填写服务器的主机名（hostname）                                  │
│                                                              │
│ ✅ 正确示例：                                                  │
│   • web-01-core                                               │
│   • db-master-primary                                         │
│   • app-gateway-dmz                                           │

│ 💡 查看命令：hostname 或 uname -n                              │
```

### 2.6 网络端口/服务

```
┌──────────────────────────────────────────────────────────────┐
│ 📝 填写说明                                                   │
│                                                              │
│ 填写服务器上开放的端口号和对应的服务名称                          │
│                                                              │
│ ✅ 正确格式：端口/服务名（逗号分隔）                             │
│   • 22/ssh, 443/https, 3306/mysql                            │
│   • 80/http, 3389/rdp, 8080/tomcat                           │
│   • 22/ssh, 9090/prometheus                                  │
│                                                              │
│ ❌ 常见错误：                                                  │
│   • "22, 80, 443" ← 缺少服务名，看不出是什么                    │
│   • "端口22" ← 不要汉字，直接写数字                            │
│   • "所有端口" ← 不可能开放所有端口，请实际查看                   │
│   • 留空 ← 至少写SSH/RDP                                     │
│                                                              │
│ 💡 查看命令：                                                  │
│   Linux: ss -tlnp 或 netstat -tlnp                            │
│   Windows: netstat -ano                                      │
└──────────────────────────────────────────────────────────────┘
```

## 3. 易错点汇总

| 序号 | 常见错误 | 正确做法 | 后果 |
|:---:|:---|:---|:---:|
| 1 | **IP为空** | 必须填写管理IP | 资产无法定位，测评无法开展 |
| 2 | **OS版本不完整** | 填写完整版本号（含小版本） | 加固方案可能选错 |
| 3 | **区域不填** | 从预制列表选择 | 无法区分内外网资产 |
| 4 | **系统名称写IP** | 写业务系统名称 | 甲方自己都看不懂哪台机器 |
| 5 | **端口不写服务** | "22/ssh" 而不是 "22" | 不知道端口上跑了什么 |
| 6 | **多台IP合在同一行** | 一台一行 | 资产无法一一对应 |
| 7 | **填写旧信息** | 先确认当前状态 | 方案针对错误版本 |
| 8 | **不适用字段留空** | 明文填"-" | 区分「未填」和「无」 |

## 4. 快速自查清单

填写完毕后，请逐项确认：

```
[✅] 所有IP地址已填写且格式正确
[✅] 所有操作系统版本包含完整版本号
[✅] 所有服务器已分配到正确的区域
[✅] 端口号列出了所有开放端口（不仅仅是22/443）
[✅] 不适用的字段填写了"-"而非留空
[✅] 系统名称是中文可读的业务名称
[✅] 每台服务器独占一行，没有合并填写
[✅] 主机名已经填写
[✅] 如果有数据库，已注明数据库版本
[✅] 如果有中间件，已注明中间件版本
```

> **最后提醒**：资产采集的质量直接决定整改方案的有效性。花 30 分钟填表，可以省掉后续 3 天的反复沟通确认。如有疑问，请联系项目负责人。
