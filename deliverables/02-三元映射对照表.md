# 交付物二：三元映射对照表

> **结论先行**：三元映射将 GB/T 22239-2019 管理要求 → CIS 技术基准 → 可执行操作命令 三条链路打通，实现「合规依据可查、技术实现可循、操作结果可验」的闭环。建议建设单位将此表作为整改实施的核心工作指引。

---

## 1. 映射结构

每条加固项的三元映射格式：

```
[GB/T 22239]  控制点.要求项号  |  要求原文（管理要求层）
[CIS]          CIS基准章节号     |  CIS建议原文（技术基准层）
[操作]         具体命令/配置路径  |  验证命令（实操执行层）
```

### 1.1 三种使用模式

| 角色 | 关注层面 | 查看重点 |
|:---|:---|:---|
| **合规负责人** | 第一重→第二重 | 管理要求是否映射到技术基准，合规闭环是否完整 |
| **技术实施** | 第二重→第三重 | 按 CIS 基准执行操作命令，完成后截图验证 |
| **测评师** | 第三重→第一重 | 操作结果是否充分证明管理要求的满足 |

## 4. 证据链说明

三元映射的核心产出不仅是加固操作本身，更在于**可交付的合规证据**：

| 证据类型 | 采集方法 | 对应三元映射环节 |
|:---|:---|:---|
| 截图（命令输出） | `script` 录制或手动截图 | 验证命令列的输出结果 |
| 配置文件快照 | `cp /etc/xxx /evidence/` | 操作命令列的配置结果 |
| 审计日志样本 | `ausearch -ts today -r` | 安全审计项的实时数据 |
| 扫描报告 | `lyn audit --severity high` | 巡检/自评的批量输出 |
| 变更记录 | `git log` 或变更管理表单 | 所有操作的可追溯性证明 |
