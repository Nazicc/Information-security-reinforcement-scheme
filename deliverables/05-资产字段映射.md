## 2. 收集表 → 规范表 转换规则

### 2.1 字段归类

ISEAA 规范表包含约 46 列，分为 5 大类：

| 大类 | 包含字段（示例） | 是否来自收集表 | 备注 |
|:---|:---|:---:|:---|
| **基本信息** | IP、系统名称、主机名、所属区域 | ✅ 直接映射 | 必填 |
| **硬件信息** | 厂商/型号、CPU、内存、磁盘 | ✅ 可选收集 | 无则填"待补充" |
| **软件信息** | OS版本、DB版本、中间件版本、应用组件 | ✅ 必收集 | 影响加固方案选择 |
| **部署信息** | 物理位置、部署方式、机房编号 | ✅ 可选收集 | 主要用于物理安全评估 |
| **管理信息** | 责任人、资产编号、标签、备注 | ✅ 可选收集 | 便于责任追踪 |

### 2.2 数据清洗规则

| 常见问题 | 处理方式 | 示例 |
|:---|:---|:---|
| IP为空 | 标记「未填写」，要求补充 | 案例报告中IP列为空 → 打回 |
| OS版本不完整 | 补充完整版本号 | "CentOS" → "CentOS 7.9.2009" |
| 端口格式不统一 | 规范化 | "80,443" → "80/http, 443/https"; "端口443" → "443/https" |
| 区域名称不一致 | 统一为项目预制枚举列表 | "核心区" → "核心业务区" |
| 同一台设备重复出现 | 去重 | 相同IP保留一个条目，合并信息 |
| 责任人格式不一致 | 规范为 "姓名(邮箱)" | "张经理" → "张三(zhangsan@xx.com)" |

## 4. 资产分组索引

根据交付物一的 5 组策略，每组资产索引规则：

| 分组 | 筛选条件 | 关键索引字段 | 加固方案适用 |
|:---|:---|:---|:---|
| G1 Linux | OS版本含 Linux | 系统名称、IP、OS版本 | linux.md（身份鉴别/访问控制/安全审计等） |
| G2 Windows | OS版本含 Windows | 系统名称、IP、OS版本 | windowsxi-tong.md（域控/防火墙/组策略） |
| G3 数据库 | 数据库版本非空 | 数据库版本、端口 | shu-ju-ku.md（数据库加固） |
| G4 中间件 | 中间件版本非空 | 中间件类型、版本 | ying-yong-xi-tong.md（Tomcat/Nginx等加固） |
| G5 网络设备 | 从资产表分离（如交换/路由） | 设备类型、管理IP | 单独方案 |

> **映射公式**：`资产所属组 = 分组筛选条件匹配结果`
> 一台设备可同时属于多组（如 Linux 服务器同时运行 MySQL → 同时属于 G1 + G3）
